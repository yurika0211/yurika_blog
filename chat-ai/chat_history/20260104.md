import numpy as np
import random

# シンプルな環境：4状態、2行動（左、右）。状態0スタート、状態3ゴール。
class SimpleEnv:
    def __init__(self):
        self.state = 0
        self.goal = 3

    def step(self, action):
        if action == 0:  # 左
            self.state = max(0, self.state - 1)
        else:  # 右
            self.state = min(3, self.state + 1)
        reward = 1 if self.state == self.goal else -0.1
        done = self.state == self.goal
        return self.state, reward, done

# エージェント
class QAgent:
    def __init__(self, num_states, num_actions):
        self.q_table = np.zeros((num_states, num_actions))
        self.alpha = 0.1  # 学習率
        self.gamma = 0.9  # 割引率
        self.epsilon = 0.1  # 探索率

    def choose_action(self, state):
        if random.uniform(0, 1) < self.epsilon:
            return random.randint(0, 1)  # ランダム行動
        else:
            return np.argmax(self.q_table[state])

    def update(self, state, action, reward, next_state):
        predict = self.q_table[state, action]
        target = reward + self.gamma * np.max(self.q_table[next_state])
        self.q_table[state, action] += self.alpha * (target - predict)

# トレーニング
env = SimpleEnv()
agent = QAgent(4, 2)
for episode in range(1000):
    env.state = 0
    done = False
    while not done:
        state = env.state
        action = agent.choose_action(state)
        next_state, reward, done = env.step(action)
        agent.update(state, action, reward, next_state)

# テスト
env.state = 0
done = False
path = [0]
while not done:
    action = agent.choose_action(env.state)
    env.step(action)
    path.append(env.state)
    done = env.state == 3
print("Path:", path)

首先，创建型模式（Creational Patterns）：这些模式关注对象的创建过程，避免直接用new关键字，增加灵活性。概率上说，新手程序员忽略这些的几率有50%以上，导致代码耦合紧。

1. 单例模式（Singleton）：确保类只有一个实例，比如数据库连接池，全局配置。实现：私有构造函数+静态getInstance方法。例子：Java的Runtime类。

2. 工厂方法模式（Factory Method）：定义创建接口，让子类决定实例化哪个类。比如图形工厂，根据类型创建圆或方。优点：扩展容易，不改原有代码。

3. 抽象工厂模式（Abstract Factory）：创建相关对象家族，而不指定具体类。比如UI套件工厂，为Windows或Mac创建按钮、窗口。适合主题切换。

4. 建造者模式（Builder）：复杂对象逐步构建，比如汉堡建造者，选肉、菜、酱分开加。例子：StringBuilder in Java。

5. 原型模式（Prototype）：通过克隆创建对象，避免昂贵初始化。比如细胞分裂模拟，克隆对象修改属性。

其次，结构型模式（Structural Patterns）：处理类和对象的组合，形成更大结构，关注接口兼容和简化。

1. 适配器模式（Adapter）：让不兼容接口合作，比如旧系统适配新API。例子：电源适配器，转圆孔到方孔。

2. 桥接模式（Bridge）：抽象与实现分离，比如形状（抽象）和颜色（实现），独立变化。适合多维度扩展。

3. 组合模式（Composite）：树形结构，如文件系统（文件+文件夹统一对待）。统一处理部分与整体。

4. 装饰器模式（Decorator）：动态添加责任，比如咖啡加奶、糖，不改原类。例子：Java的InputStream装饰。

5. 外观模式（Facade）：简化复杂子系统接口，比如家居智能系统，一键“回家模式”控制灯、空调。

6. 享元模式（Flyweight）：共享细粒度对象，节省内存。比如字符串池，相同字符串复用。

7. 代理模式（Proxy）：控制访问，比如远程代理（RPC）或虚拟代理（懒加载图像）。

最后，行为型模式（Behavioral Patterns）：关注对象间责任分配和通信，强调算法和交互。

1. 责任链模式（Chain of Responsibility）：请求沿链传递，比如日志级别处理。避免发送者和接收者耦合。

2. 命令模式（Command）：封装请求为对象，比如菜单命令，可撤销。例子：编辑器的undo。

3. 解释器模式（Interpreter）：定义语言语法，比如SQL解析器。适合简单语言。

4. 迭代器模式（Iterator）：遍历集合而不暴露内部，比如Java的Iterator。

5. 中介者模式（Mediator）：对象通过中介通信，比如聊天室中介，避免多对多混乱。

6. 备忘录模式（Memento）：保存/恢复状态，比如游戏存档，不暴露内部。

7. 观察者模式（Observer）：一对多依赖，状态变通知观察者。比如事件订阅。

8. 状态模式（State）：对象行为随状态变，比如TCP连接状态（listen、established）。

9. 策略模式（Strategy）：算法可互换，比如排序策略（冒泡、快速）。

10. 模板方法模式（Template Method）：定义骨架，子类实现步骤。比如泡茶模板： boilWater → brew → pour → addCondiments。

11. 访问者模式（Visitor）：操作元素而不改类结构，比如文件访问者统计大小。
